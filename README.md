# МАБЛ · LMS-платформа

LMS-сайт **Международной академии бизнес лидерства (МАБЛ)** на React + Vite +
TypeScript. Визуальная айдентика строго по бренд-гайду `MABL_GUIDE.pdf`:
цвета Нефть / Океан / Мудрость, герб и логотип академии, шрифт TT Rationalist,
академический премиум-минимализм.

## Технологии

- **React 18** + **TypeScript** (strict)
- **Vite 5**
- **React Router 6**
- **Tailwind CSS 3** (брендовые токены в `tailwind.config.js`)
- **Mock-данные** вместо бэкенда (`src/data/*`)

## Быстрый старт

```bash
npm install
npm run dev      # http://localhost:5173
npm run build    # сборка в dist/
npm run preview  # предпросмотр прод-сборки
```

### Шрифт TT Rationalist

TT Rationalist — коммерческий шрифт (TypeType), файлы не включены в репозиторий
по лицензии. Положите лицензионные `.woff2` в `public/fonts/` с именами из
`src/styles/fonts.css` (см. `public/fonts/README.md`). До этого используется
серифный fallback (Georgia), близкий по строю — вёрстка остаётся академичной.

### Платежи · ЮKassa (готово, включается переменной)

UI работает с интерфейсом `PaymentProvider` (`src/lib/payments.ts`) и не знает о
конкретном шлюзе. Активный провайдер выбирается переменной `VITE_PAYMENT_PROVIDER`:

| Значение | Поведение |
|----------|-----------|
| `mock` (по умолчанию) | Демо-оплата: ничего не списывает, доступ открывается сразу. |
| `yookassa` | Боевая оплата ЮKassa с редиректом на платёжную форму. |

**Интеграция ЮKassa уже реализована и «спит», пока не заданы ключи** — это
позволяет подключить её позже, после подготовки курса, без правок кода:

1. В личном кабинете ЮKassa (Магазин → Интеграция) возьмите `shopId` и секретный
   ключ. Пропишите серверные переменные на Vercel:
   `YOOKASSA_SHOP_ID`, `YOOKASSA_SECRET_KEY` (секреты во фронтенд не попадают).
2. В настройках ЮKassa укажите URL для уведомлений (webhook):
   `https://<домен>/api/payments/webhook` — события `payment.succeeded` и
   `payment.canceled`.
3. Включите боевой провайдер на фронтенде: `VITE_PAYMENT_PROVIDER=yookassa`.

Поток оплаты (всё проверяется сервером, клиент не диктует цену):

- `POST /api/payments/create` — заводит заказ (`pending`) в БД и создаёт платёж
  в ЮKassa с ключом идемпотентности; возвращает ссылку на платёжную форму.
- Пользователь оплачивает на стороне ЮKassa и возвращается на
  `/checkout?course=…&order=…`, где статус подтверждается через
  `GET /api/payments/by-order/<id>` (перезапрос платежа в ЮKassa).
- `POST /api/payments/webhook` — серверное уведомление: статус заказа в БД
  переводится в `paid` / `refunded` (тело не доверяется — платёж перезапрашивается
  по id).
- `GET /api/payments/config` — отдаёт, настроена ли боевая оплата.

Проверить готовность можно тестовым магазином ЮKassa (тестовый `shopId`/ключ).

## Новости из Telegram-канала

Раздел «Новости» автоматически подтягивает публикации (текст + изображения) из
публичного Telegram-канала академии. Источник — веб-превью `t.me/s/<канал>`,
поэтому бот и токен не нужны — достаточно, чтобы канал был **публичным**.

Настройка (переменные окружения проекта на Vercel):

| Переменная | Назначение |
|------------|------------|
| `TELEGRAM_CHANNEL` | Публичный `@username` канала (можно без «@» или ссылкой) |
| `DATABASE_URL` | Подключение к Neon (новости хранятся в таблице `news`) |

Как это работает:

- **Cron на Vercel** (`vercel.json` → `crons`) раз в сутки (06:00 UTC) дёргает
  `GET /api/news/sync`, который читает последние посты канала и делает
  идемпотентный upsert в таблицу `news` (id вида `tg-<канал>-<id поста>` —
  повторы не дублируются, отредактированные посты обновляются).
- **Кнопка «↻ Обновить из Telegram»** в `/admin/news` запускает ту же
  синхронизацию вручную (`POST /api/news/sync`).
- Публичный раздел `/news` и API `GET /api/news` отдают новости из БД,
  отсортированные по дате публикации; обложки постов показываются на карточках
  и на странице новости.

> Тариф Vercel Hobby разрешает cron-задачи только раз в сутки, поэтому расписание
> стоит на 06:00 UTC. Для более частого обновления используйте кнопку «Обновить
> из Telegram» в админке или тариф Pro (там можно вернуть, например, `0 * * * *`).

Новости, созданные вручную в админ-панели (`source = manual`), и импортированные
из Telegram (`source = telegram`) живут в одной таблице и не конфликтуют.

## Деплой на Vercel

В репозитории есть `vercel.json` (framework: vite, SPA-rewrites). Варианты:

1. **Через дашборд:** импортируйте репозиторий на vercel.com → Vercel сам
   определит Vite. Build: `npm run build`, Output: `dist`.
2. **Через CLI:**
   ```bash
   npm i -g vercel
   vercel        # preview-деплой
   vercel --prod # продакшен
   ```

SPA-роутинг покрыт rewrite-правилом на `index.html`.
